Ключевые кадры @keyframes

Ключевые кадры @keyframes позволяет менять CSS стили для заданного селектора в любой точке в момент смены состояния или события. Структура выглядит вот так:


Если код не виден, вы можете скролить серый блок с кодом.


@keyframes название анимации { ...% { ... } ...% { ... } /* Можно писать еще ключевые кадры */ }

Название анимации это свойство animation-name, увидите чуть ниже, а проценты представляют собой точки вдоль временной линии анимации, внутри которых задаются стили. Например в точке изначального положения(старта), 0%, стили такие, а в конечной точке, 100%, стили вот такие. Вместо 0% и 100% можно использовать ключевые слова соотвественно from и to. Теперь давайте поговорим об анимации.



Свойство animation

Свойство animation позволяет сделать анимацию элементу. Свойство animation нужно писать до ключевых кадров @keyframes. Свойство не наследуется. Его можно разделить еще на несколько свойств:


1) animation-name

Свойство animation-name задает название анимации. Название может быть любым.


2) animation-duration

Свойство animation-duration задает длительность анимации. Значения: s - секунда, ms - миллисекунда. 1s = 1000ms.


3) animation-timing-function

Свойство animation-timing-function указывает как быстро по времени меняется анимация. Значения такие же как у свойства transition как ease, linear, ease-in и т.д.


4) animation-iteration-count

Свойство animation-iteration-count задает количество повторов анимации. Значения: натуральные числа или значение infinite(бесконечно).


5) animation-direction

Свойство animation-direction определяет тип движения ключевых кадров. Значения:


normal - как обычно. Это значение по умолчанию.


alternate - каждый четный проход по клоючевым кадрам будет проигрываться в обратную сторону.


alternate-reverse - каждый нечетный проход по ключевым кадрам будет проигрываться в обратную сторону.


reverse - задом наперет.


5) animation-play-state

Свойство animation-play-state позволяет приостановить анимацию, например при наведении. Значения:


running - анимация идет. Это значение по умолчанию.


paused - остановить анимацию.


6) animation-delay

Свойство animation-delay позволяет сделать паузу до начала анимации. Значения: секунды и миллисекунды.


7) animation-fill-mode

Свойство animation-fill-mode позволяет остановить анимацию в конечном ключевом кадре. Значения: none - не остановить(значение по умолчанию), forwards - остановить в конечном ключевом кадре.



Можно всё упростить и писать значения этих свойство в одно свойство animation:

... { animation: 1) 2) 3) 4) 5) 6) 7); }

Можно не писать значения свойств которые по умолчанию, а просто продолжить писать значения следующих нужных нам свойств.


Например:
HTML код:
<body> <div id="blocktough"> <span>Я кручусь и я сумасшедший</span> </div> </body>
CSS код:
#blocktough { width: 150px; text-align: center; padding: 50px 0; margin: 20px auto; background-color: #3d77cf; -webkit-animation: crazyblock 3s linear infinite alternate; animation: crazyblock 3s linear infinite; } @keyframes crazyblock { 0% { -webkit-transform: rotate(0deg); -ms-transform: rotate(0deg); transform: rotate(0deg); } 100% { -webkit-transform: rotate(360deg); -ms-transform: rotate(360deg); transform: rotate(360deg); } }

Я кручусь и я сумасшедший


Правило/Медиа запросы @media

Правило @media используется для изменения CSS стилей на основании определенных характеристик устройства, связанных с отображением контента, таких как ширина, высота, ориентация, разрешение экрана и так далее. Оно позволяет адаптировать контент под разные мобильные устройства. Например на компьютере и телефоне, один и тот же сайт будет выглядить по другому.


Также на уроке HTML мы говорили о мета теге который пропишем при адаптивности сайта. Так вот этот мета тег выглядить вот так:

<meta name="viewport" content="width=device-width, initial-scale=1">

Атрибут name предназначен для того чтобы указать браузеру, какую именно информацию о странице хотим ему сообщить. В данном случае эта информация касается viewport.


Контент (содержимое) этих сведений указывается в качестве значения атрибута content посредством пар ключ-значение, разделённых между собой запятыми.


Для адаптивного дизайна значения атрибута content viewport должно определяться 2 параметрами:


width=device-width

initial-scale=1


После добавляния мета тега, вы готовы приступить к медиа запросам.


Структура медиа запросов выглядит так:

@media ... { /* Стили */ }

Под тремя точками можно писать тип устройства(необязательно), условие которое связано с характеристикой устройства. Чаще всего под условием выступает максимальная или минимальная ширина устройства. Условие надо писать в скобках и без точки с запятой после значения свойства. Например у нас есть блок с идентификатором #block, и нам нужно чтобы он до 500 пикселей в ширину экрана не появлялся и исчез. Всё будет выглядить вот так:

#block { display: block } @media (max-width: 500px) { #block { display: none; } }

Также к медиа запросам можно добавлять логические операторы. С помощью них можно делать комбинированные медиазапросы, в которых будет проверяться соответствие нескольким условиям.


Оператор and

Оператор and связывает друг с другом разные условия:

@media (min-width: 200px) and (max-width: 600px) { /* Стили */ }


Доп. материал

Есть специальные свойства которые прописывают только к определенным селекторам. Их мы щас рассмотрим:


Свойство list-style

Это свойство используется для тегов ol или ul. Для него можно прописать еще ключевые слова через дефис, например list-style-position, которое определяет, как будет размещаться маркер(точки) относительно текста. Вы также можете при надобности поискать это свойство и его значения в интернете. Свойство наследуется.


Свойство outline

Это свойство одновременно устанавливающее цвет, стиль и толщину внешней границы на всех четырех сторонах элемента. Иногда применяется для input, чтобы когда мы нажали, не было черной полосы на всех четырех сторонах элемента, для этого пишется значение 0. Также можно писать ключевые слова свойству через дефис. Их также можно поискать в интернете. Свойство не наследуется.


Например:
HTML код:
<body> <div> <span>До:</span><br> <input type="text" placeholder="Введите что-то"> </div> <br> <div> <span>После:</span><br> <input id="input2" type="text" placeholder="Введите что-то"> </div> </body>
CSS код:
#input2 { outline: 0; }

До:

После:


Псевдоэлемент ::marker

Псевдоэлемент ::marker позволяет управлять маркерами(точками) у элементов списка. Вот например:

HTML код:
<body> <ul> <li>Привет!</li> <li>Мир!</li> </ul> </body>
CSS код:
ul { list-style-position: inside; } ul li::marker { color: blue; }

  • Привет!
  • Мир!

Свойство cursor

Свойство cursor мыши позволяет изменять курсор мыши при наведении. Есть куча значений, их вы можете поискать в интернете.


Например:
HTML код:
<body> <p>Наведи на меня</p> </body>
CSS код:
p:hover { cursor: pointer; }

Наведи на меня



Правило !important

!important играет роль, когда пользователь хочет подключить/изменить таблицу стилей. Если у автора в таблице стилей написан !important, то сработают его стили, а пользователь не сможет их изменить. Грубо говоря, он повышает приоритет стиля где он написан. Но, есть и другие исключения:


!important добавлен в авторский стиль — будет применяться стиль автора.


!important добавлен в пользовательский стиль — будет применяться стиль пользователя.


!important нет как в авторском стиле, так и стиле пользователя — будет применяться стиль автора.


!important содержится в авторском стиле и стиле пользователя — будет применяться стиль пользователя.


Структура выглядит вот так:

Селектор { свойство: значение !important; }

Таблицу стилей пользователь может изменять в исходном коде страницы.
