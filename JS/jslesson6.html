Отслеживать JS атрибуты не в HTML

Мы использовали JavaScript атрибуты по типу onclick, oninput и т.д. Но есть более удобный метод это способ как отслеживать объекты HTML, это метод addEventListener(). Этот метод позволяет добавлять обработчики событии и их значения не трогая при этом сам HTML документ.


Метод addEventListener как и все остальные методы прописывается сразу после выбора объекта если у нас только одно действие, или после переменной.


В ней пишут два параметра: первое - атрибут, второе - значение атрибута(функция). Атрибут пишется в кавычках и без приставки on или разных других.


Если код не виден, вы можете скролить серый блок с кодом.


HTML код:
<body> <button id="buttonclick"></button> </body>
JS код:
document.querySelector("#buttonclick").addEventListener("click", buttonClick); function buttonClick() { ... } // Или let button = document.querySelector("#buttonclick"); button.addEventListener("click", buttonClick); function buttonClick() { ... }

Но можно не писать название функции как второе значение а сразу писать функцию там:

document.querySelector("#buttonclick").addEventListener("click", function() { ... });


Внимание! При написании метода addEventListener, у вас может появится ошибка Uncaught TypeError: Cannot read property 'addEventListener' of null. Можете поискать эту ошибку в интернете. У всех причина возникновения может быть разной. Но чаще всего решением оказывается просто перетащить сам код скрипта внутри HTML документа перед закрывающем тегом body.



Работа с формой

Помните мы говорили на уроке HTML что мы поработаем над кнопкой submit? Так вот, эта кнопка отправляет данные на сервер. Но так как это JavaScript, и мы не сможем здесь работать с сервером, мы просто сделаем аналогию отправки данных. Никакой отправки данных здесь не будет.


input и button с типами submit, нету разницы. Можно использовать и то и другое.


Давайте создадим простую форму:

HTML код:
<body> <form method="post" id="formsubmit"> <label for="name">Введите ваше имя</label> <input type="text" placeholder="Имя" name="name" id="name"><br><br> <label for="password">Пароль</label> <input type="password"placeholder="Пароль" name="password" id="password"><br><br> <label for="repassword">Повтор пароля</label> <input type="password" placeholder="Повтор пароля" name="repassword" id="repassword"><br><br> <div id="mistakes"></div> <button type="submit">Готово</button> </form> </body>
JS код:
document.querySelector("#forsubmit").addEventListener("submit", buttonSubmit); function buttonSubmit() { }

Когда мы нажимаем на кнопку с типом submit, у нас отправляются данные на сервер и перезагружается страница. Чтобы отключить перезагрузку страницы при нажатии на эту кнопку, делаем вот что: Если мы вместо addEventListener добавляем обработчик события onsubmit в тег form, то чтобы отключить перезагрузку страницы, используем оператор return.


return false; - не перезагружается страница. Можете представлять это как что false = перезагрузка. То есть мы вытащили этот false из функции, и страница не перезагрузилась.

return true; - перезагружается страница. Можете представлять это как что true = не перезагрузка. То есть мы вытащили этот true из функции, и страница перезагрузилась.


HTML код:
<body> <form method="post" id="formsubmit" onsubmit="exmpSubmit(this)"> <label for="name">Введите ваше имя</label> <input type="text" placeholder="Имя" name="name" id="name"><br><br> <label for="password">Пароль</label> <input type="password" placeholder="Пароль" name="password" id="password"><br><br> <label for="repassword">Повтор пароля</label> <input type="password" placeholder="Повтор пароля" name="repassword" id="repassword"><br><br> <div id="mistakes"></div> <button type="submit">Готово</button> </form> </body>
JS код:
// Если бы мы использовали обработчик события let objform = document.querySelector("#formsubmit"); function exmpSubmit() { return false; }

Если мы протестируем это, ты видим что страница все таки перезагружается. Почему? Дело в том что если эта запись с return не связана с обработчиком событии. В мысли сразу приходит идея сделать подобным образом:

// Если бы мы использовали обработчик события let objform = document.querySelector("#formsubmit"); function exmpSubmit(el) { el.return false; }

Но к сожалению эта конструкция не работает. Чтобы сделать это, мы должны в значении обработчика событии тоже добавить оператор return:

<body> <form method="post" id="formsubmit" onsubmit="return exmpSubmit(this)"> <label for="name">Введите ваше имя</label> <input type="text" placeholder="Имя" name="name" id="name"><br><br> <label for="password">Пароль</label> <input type="password" placeholder="Пароль" name="password" id="password"><br><br> <label for="repassword">Повтор пароля</label> <input type="password" placeholder="Повтор пароля" name="repassword" id="repassword"><br><br> <div id="mistakes"></div> <button type="submit">Готово</button> </form> </body>

Теперь форма будет принимать результат значения функции и будет не перезагружать страницу.




Чтобы отключить перезагрузку страницы если мы используем вместо обработчиков события, addEventListener, то мы обращаемся к параметру функции обращенной к тегу form, и добавляем такой метод как preventDefault(). return уже не прописываем ибо не прописан атрибут события в теге form.

HTML код:
<body> <form method="post" id="formsubmit"> <label for="name">Введите ваше имя</label> <input type="text" placeholder="Имя" name="name" id="name"><br><br> <label for="password">Пароль</label> <input type="password" placeholder="Пароль" name="password" id="password"><br><br> <label for="repassword">Повтор пароля</label> <input type="password" placeholder="Повтор пароля" name="repassword" id="repassword"><br><br> <div id="mistakes"></div> <button type="submit">Готово</button> </form> </body>
JS код:
document.querySelector("#formsubmit").addEventListener("submit", buttonSubmit); let mistakes = document.querySelector("#mistakes"); mistakes.style.cssText = "color: red;"; function buttonSubmit(event) { let objform = document.querySelector("#formsubmit"); // Находится тег form event.preventDefault(); }


Перезагрузку мы отключили. Теперь давайте теперь будем отслеживать значения input'ов. Для этого будем обращаться к ним через атрибуты:

HTML код:
<body> <form method="post" id="formsubmit"> <label for="name">Введите ваше имя</label> <input type="text" placeholder="Имя" name="name" id="name"><br><br> <label for="password">Пароль</label> <input type="password" placeholder="Пароль" name="password" id="password"><br><br> <label for="repassword">Повтор пароля</label> <input type="password" placeholder="Повтор пароля" name="repassword" id="repassword"><br><br> <div id="mistakes"></div> <button type="submit">Готово</button> </form> </body>
JS код:
document.querySelector("#formsubmit").addEventListener("submit", buttonSubmit); let mistakes = document.querySelector("#mistakes"); mistakes.style.cssText = "color: red;"; function buttonSubmit(event) { let objform = document.querySelector("#formsubmit"); let name = objform.name.value; let password = objform.password.value; let repassword = objform.repassword.value; event.preventDefault(); }

Пропишем пару условии:

document.querySelector("#formsubmit").addEventListener("submit", buttonSubmit); let mistakes = document.querySelector("#mistakes"); mistakes.style.cssText = "color: red;"; function buttonSubmit(event) { let objform = document.querySelector("#formsubmit"); let name = objform.name.value; let password = objform.password.value; let repassword = objform.repassword.value; if(name === "" || password === "" || repassword === "") { mistakes.innerHTML = "Заполните все поля"; } else if(password != repassword) { mistakes.innerHTML = "Пароли должны совпадать"; } else { mistakes.innerHTML = ""; alert("Данные успешно отправлены"); } event.preventDefault(); }








Опять же, никакие данные никуда не отправляются, так как это не серверный язык. Можете свободно писать что-либо.



Загрузка DOM, загрузка окна

DOM(Document Object Model) - это весь HTML документ, все объекты, все теги. Когда мы хотим чтобы выполнялась операция JavaScript после загрузки всех объектов HTML, можем прописать такую конструкцию(но внешние ресурсы, например картины, еще могут быть не загружены):

document.addEventListener("DOMContentLoaded", function() { // Код });

Также есть такой глобальный объект который называется window. Оно позволяет обращаться к окну браузера.

Если мы хотим чтобы операция JavaScript выполнялась после загрузки всего окна, даже внешних ресурсов по типу картинок, то можем использовать такую конструкцию:

window.onload = function() { // Код }


Интервалы

Интервалы - позволяют выполнять код через определенные промежутки времени постоянно. Структура:

setInterval(название функции, время); function название функции() { ... }

Или такая структура:

setInterval(function() { ... }, время);

Время надо писать в миллисекундах только без ключевого слова ms. 1 секунда = 1000 миллисекундам.

Например:
let count = 0; setInterval(function() { count++; console.log(count); }, 1500);

Чтобы остановить интервал, используют функцию clearInterval(). Но для начала давайте отметим наш интервал поместив его в переменную:

let count = 0; let int = setInterval(intcount, 1000); function intcount() { count++; if(count >= 15) clearInterval(int); // Передаем интервал console.log(count); }


Таймеры

Таймеры - позволяет сработать коду только один раз через определенный промежуток времени. Структура точно такая же как у интервалов:

setTimeout(название функции, время); function название функции() { ... } // Или setTimeout(function() { ... }, время);

Пример:
setTimeout(function() { console.log("Я появился через 3 секунды") }, 3000);
