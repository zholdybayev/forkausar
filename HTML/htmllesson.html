Что такое HTML?


Введение в HTML

HTML — HyperText Markup Language — это язык гипертекстовой разметки страницы. По сути, уникальная вещь позволяющая дать браузеру понять как надо отображать нужные объекты. Под гипертекстом в этом случае понимается ТЕКСТ, связанный с другими текстами указателями-ссылками. То есть HTML работает над текстами, уже сами стили это уже язык CSS. О нем поговорим в другом уроке.


После того, как мы поняли что такое HTML, давайте посмотрим основной вид HTML документа(Базовая структура HTML кода). Эта структура ОБЪЯЗАТЕЛЬНА. Она одинакова на всех сайтах и её пишут самой первой при написании кода.


Если код не виден, вы можете скролить серый блок с кодом.


<!DOCTYPE html> <html> <head> <!-- Здесь будет метаинформация сайта --> </head> <body> <!-- Здесь будет само тело сайта --> </body> </html>

Именно вот так выглядит базовая структура HTML кода. Можете просто всегда копировать и вставлять это в свой HTML документ.

Внутри тега head мы пишем метаинформацию сайта: подключаем разные документы, указываем кодировку и т.п.

Внутри тега body уже будет само тело сайта, сами объекты на экране.


Теперь давайте поговорим о самих тегах.



Теги в HTML

Теги в HTML это своеобразные команды которые говорят браузеру какие объекты надо вставлять а какие нет. Теги бывают двух типов: одинарные и парные: Одинарные записываются следующим образом:

<Тег>

А парные вот так:

<Тег></Тег>

Как вы уже видите, отличие между одинарными тегами это в том что у второго "закрывающего" тега перед самим названием тега стоит "/".


Обычно если тег парный, то в нём можно записать текст:

<Тег>Какой-то текст</Тег>

Также можно писать теги в тегах:

<Тег> <Тег></Тег> </Тег>

Или вот так:

<Тег><Тег></Тег></Тег>

Но мы рекомендуем писать первым вариантом, ибо вы в будущем запутаетесь в своем же коде.



Тег который находится внутри другого тега называют дочерним элементом. Тот который снаружи дочернего элемента называют родительным элементом. Так называется структура отношений тегов документа между собой.

<Родительный тег> <Дочерний тег></Дочерний тег> </Родительный тег>

Для тегов можно добавлять еще атрибуты. О них и поговорим ниже.



Атрибуты

Атрибутов как и тегов тоже много. Они определяют разные характеристики тегов. Атрибуты пишутся в тегах и в них записывают значения атрибута. Выглядит всё следующим образом:

<Тег атрибут="значение">

Атрибутов вы можете добавлять сколько угодно при помощи пробела:

<Тег атрибут1="значение" атрибут2="значение" атрибут3="значение">


Давайте рассмотрим один пример. У нас есть тег "Человек". И мы должны задать ему характеристики пола, веса, роста. Все будет выглядеть следующим образом:

<Человек пол="мужской" вес="71кг" рост="175см">


Теперь можем остановится на самих реальных тегах и атрибутах.


Атрибуты которые можно вставлять во все теги


Атрибут class

Атрибут class задает стилевой класс, который позволяет связать определенный тег со стилевым оформлением. При помощи class'ов мы можем обращаться к объектам в CSS и JS. Значение у class'а может быть каким угодно только не начиная с цифр или не имея спец. символ.

<Тег class="box">Я коробка с классом box!</Тег>

Значение атрибута class у нескольких элементов может быть одинаковым. А это значит, что в будущем мы сможем обращаться к нескольким объектам одновременно.

<Тег class="text">У меня класс text :)</Тег> <Тег class="text">У меня тоже класс text</Тег> <Тег class="text">У меня тоже</Тег>

К любому тегу одновременно можно добавить несколько классов, перечисляя их в атрибуте class через пробел.

<Тег class="text example">У меня класс text и example!</Тег>


Атрибут id

Атрибут идентификатор id почти как class, только с некоторыми исключениями. В коде документа каждый идентификатор уникален и должен быть включён лишь один раз. То есть атрибут id с определенным значением должен быть только один в коде. Значение у id может быть каким угодно только не начиная с цифр или не имея спец. символ.

<Тег id="ident">У меня идентификатор ident и ни у другого тега нету такого id</Тег>

Атрибут id также позволяет связать определенный тег со стилевым оформлением. При помощи id мы можем обращаться только к определенному объекту в CSS и JS.


Значение атрибута id у нескольких элементов не может быть одинаковым.


К тегу id нельзя добавить несколько значений через пробел.

i

Примечание

Значения атрибутов id и class(да и вообще любые придуманные значения атрибутов) чувствительны к регистру. Например класс someclass и someClass это два разных класса.

По сути, атрибуты class и id безполезны в HTML без CSS. Так что мы подробно остановимся на них в уроке CSS.



Атрибут title

Атрибут title позволяет показывать подсказку при наведений на объект.

<Тег title="Привет! Я подсказка">Наведи на меня</Тег>

Теги и атрибуты внутри тега head


Тег meta

Тег meta обеспечивает метаданные о HTML документе. Метаданные не отображаются на странице, но интерпретируются браузерами и поисковыми системами. Мета элементы, как правило, используются, чтобы указать описание страницы, ключевые слова, автора документа и другие метаданные. Закрывающий тег не имеется.

<meta>

При каждом написании или изменении кода в редакторе кода, чтобы увидеть измененный результат на странице, надо сохранить измененные действия в редакторе кода и обновить страницу.


У тега meta есть несколько атрибутов. Основными являются http-equiv, name, content.

<meta http-equiv="Content-Type" content="text/html" charset="UTF-8">

Атрибут http-equiv указывает на каких основах данных следует обрабатывать основное содержание документа. Рекомендуем в значении писать Content-Type. Значит здесь мы будем указывать что в мета теге будем заниматься с типом содержания Content-Type.


Атрибут name это информационное имя мета тега. Оно применяется в паре с атрибутом content.


Атрибут content содержит информационное содержание. То есть здесь мы будем работать с текстом(text/html) с типом содержания Content-Type.


Атрибут charset указывает кодировку документа. Рекомендуем писать значение UTF-8.


Рекомендуем просто скопировать и вставить это в свой тег head.


<meta name="keywords" content=""> <meta name="description" content="">

Здесь значение keywords, это ключевые слова веб-страницы соответствующие содержимому страницы сайта, предназначены для поисковых машин. Внутри content надо писать ключевые слова веб-страницы.


C description тоже самое. Это краткое описание страницы. Данное описание частенько используется поисковыми системами для вывода в результатах поиска, по какому либо запросу, информации о сайте и его назначении. Внутри content надо писать краткое описание страницы, от 100 до 200 символов.

Тег title

Тег title – это элемент разметки страницы, позволяющий поисковикам выделять суть содержимого страницы. Он является заголовком окна браузера.

<head> <title>Название веб-страницы</title> </head>

Вот заголовок Google:


Title первое, что видит человек в результатах поисковой выдачи, на вкладке браузера и при перепосте ссылки на страницу в соцсетях. Оно одно из важнейших элементов страницы.



Тег link

Тег link определяет отношение между текущим html-документом и внешним ресурсом, на который он ссылается. Чаще всего тег link используется для подключения внешних таблиц стилей, вроде CSS и других. Пока мы не будем подключать таблицы стилей, просто знайте что в будущем это сыграет свою роль. Тег одинарный.



Теги и атрибуты в теге body


Комментарий

Давайте быстренько пройдемся по комментариям. Оно используется для написания комментария в коде HTML документа. HTML комментарии призваны улучшить читабельность кода, делать сообщения, уведомления в коде. Их можно писать куда-угодно в HTML документе. В комментариях обычно указывается объяснение участка кода, что упрощает процесс редактирования HTML страницы в дальнейшем. Комментарии в HTML не отображаются на странице в браузере пользователю, они игнорируются браузером при читании кода, однако их можно увидеть в исходном коде веб-страницы.

<!-- Я комментарий -->


Тег div

Тег div создает блок. То есть блок занимает целую доступную ширину. Если объект является блоком, то последующие элементы будет перенесены в строчку вниз т.к. объект занимает всю доступную ширину экрана. Если объект не является блоком, то он будет объеденён с другим объектом в одну линию.


Вот пример с блоком:

<div>Я блок и я занимаю целую эту строку</div> <span>Я был перенесен в строчку вниз</span>

Я блок и я занимаю целую эту строку
Я был перенесен в строчку вниз

Вот пример без блока:

<span>Я не блок</span> <span>Я объединен</span>

Я не блокЯ объединен

Тег span

Тег span он абсолютно ничего не изменяет в тексте. В нём просто можно писать текст. Удобно, не так ведь?


В основном он используется дабы создать кодовую метку(добавляя атрибут class или id), а затем модифицировать этот отрезак текста в CSS или JS.



Тег p

Тег p делает параграфы. Элемент является блочным, значит его содержимое всегда начинается с новой строки и занимает всю доступную ширину. В них обычно пишут одно или несколько предложений. Например эти предложения находятся в теге p.



Тег a

Тег a создает ссылки. Используется для перехода с одной страницы на другую. У тега a есть объязательный атрибут href который указывает на назначение ссылки.


Давайте сделаем пример с нажатием на текст и переходом в Google в этой же странице:

<a href="https://www.google.com/">Нажми на меня</a>

Нажми на меня


Чтобы открыть ссылку в новом окне, надо к тегу a добавить атрибут target со значением _blank.

<a href="https://www.google.com/" target="_blank">Нажми на меня</a>
